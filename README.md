# Agentic Solve Math

Корневой workspace для двух связанных направлений:

- `Math_Hypothese_AutoCheck_Witch_Agents/` — математическое исследование, долговременная память, гипотезы, доказательства, Lean и мультимодельный анализ.
- `CayleyPy_Pancake/` — инженерный проект анализа pancake-проблемы: солверы, эвристики, RL, эксперименты, логи, статья и гипотезы по улучшению качества решений.

Корень этого workspace теперь служит **оркестратором агентной работы**. Здесь хранится:

- общая карта проектов;
- долговременная память агента;
- журнал пользовательских промптов;
- правила Cursor для корневого уровня;
- канонический intake через `kaggle_intake/`.

## Канонический вход

Будущий агентный pipeline должен стартовать от файла в `kaggle_intake/` в корне workspace.

Ожидаемый сценарий:

1. Пользователь кладёт в `kaggle_intake/` markdown-файл со ссылками на Kaggle-соревнования.
2. Агент извлекает требования соревнования, внешние источники, литературу, кодовые референсы и идеи для baseline.
3. Агент распределяет работу между:
   - `ML in Math/` для инженерных экспериментов и Kaggle-пайплайна;
   - `Math_Hypothese_AutoCheck_Witch_Agents/` для математических гипотез, формализации и исследовательской памяти;
   - корневыми журналами и индексами для долгосрочной трассируемости.

Подробно:

- `BASELINE_INTAKE_SPEC.md`
- `KAGGLE_AUTONOMOUS_WORKFLOW.md`
- `SOURCE_COLLECTION_POLICY.md`

## Корневые документы

- `AGENTS.md` — главный долговременный контракт агента на уровне всего workspace.
- `PROJECT_MAP.md` — оперативная карта двух проектов и их ролей.
- `RESEARCH_SURVEY.md` — подробный аудит обоих проектов.
- `DOCUMENTATION_INDEX.md` — единая навигация по корневым и дочерним документам.
- `USER_PROMPTS_LOG.md` — полный журнал пользовательских постановок и задач.
- `AGENT_ACTIVITY_LOG_SPEC.md` — стандарт сверхподробного логирования действий агента.
- `RESEARCH_JOURNAL.md` — живой исследовательский журнал workspace.

## Принцип работы

Корень не заменяет локальную документацию дочерних проектов. Он:

- связывает их общими правилами;
- задаёт единый формат памяти и отчётности;
- фиксирует пользовательские промпты и решения агента;
- упрощает передачу контекста между сессиями и агентами;
- поддерживает подготовку материала для будущей научной работы.
