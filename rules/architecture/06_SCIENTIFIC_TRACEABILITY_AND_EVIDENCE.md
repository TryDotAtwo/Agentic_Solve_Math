# Scientific Traceability And Evidence

## Principle

Любой серьёзный вывод должен быть восстановим спустя месяцы.

## Required evidence objects

- user prompt record;
- intake file;
- official contract capture;
- source catalog;
- experiment report;
- theory note;
- decision record;
- submission record;
- canonical summary.

## Evidence ladder

1. official competition files and rules;
2. reproducible local code and logs;
3. scientific references;
4. strong external baselines;
5. discussions and informal notes;
6. agent intuition.

## Mandatory status tags

- `confirmed`
- `partial`
- `needs_verification`
- `rejected`

## Rule of separation

Нельзя смешивать в одной формулировке:

- факт;
- вывод;
- гипотезу;
- решение.

## Cross-level traceability

For every major task we should be able to reconstruct:

- what the user asked;
- what root decided;
- which subproject owned execution;
- what local team did;
- what evidence supports the final claim.

## Publication-readiness rule

Если материал может потенциально войти в статью, у него должны быть:

- provenance;
- date;
- scope;
- evidence class;
- limitations;
- canonical location.
