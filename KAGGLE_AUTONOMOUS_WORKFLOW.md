# Kaggle Autonomous Workflow

## Цель

Этот workflow описывает, как агент должен действовать после появления нового markdown-файла в корневом `kaggle_intake/`.

## Фаза 1. Intake

1. Прочитать intake-файл из `kaggle_intake/`.
2. Извлечь все competition links и дополнительные ссылки.
3. Зафиксировать:
   - имя задачи;
   - источник постановки;
   - дату старта исследования;
   - связанный пользовательский запрос.
4. Обновить:
   - `USER_PROMPTS_LOG.md`
   - `RESEARCH_JOURNAL.md`

## Фаза 2. Problem Brief

Агент должен собрать краткий, но точный brief:

- что требуется предсказывать или оптимизировать;
- какая evaluation metric;
- какой submission format;
- есть ли ограничения по времени, вычислениям, внешним данным;
- какие baseline resources уже известны.

Если часть информации неизвестна, это должно быть явно записано как `needs verification`.

## Фаза 3. Source Collection

Агент собирает:

- competition overview;
- official data description;
- rules;
- discussion threads;
- public notebooks;
- статьи и книги по теме;
- релевантные open-source repos;
- математические или алгоритмические аналоги задачи.

Сбор должен подчиняться `SOURCE_COLLECTION_POLICY.md`.

## Фаза 4. Разделение на две ветки

### Engineering branch

Направляется в `CayleyPy_Pancake/` или соответствующий CayleyPy-подпроект (например, `CayleyPy_444_Cube/`), если нужно:

- построить baseline solver;
- **первым делом** скачать требования к сабмиту и пример submission (формат колонок, типы данных);
- **обязательно** настроить и проверить автосабмит (Kaggle CLI или HTTP-обход) на тестовом submission;
- скачать и зафиксировать необходимые для анализа структуры данных (перестановки, графы и т.п.);
- экспериментировать с эвристиками, ML, search, RL;
- готовить submission;
- сравнивать score;
- автоматизировать pipelines.

### Mathematics branch

Направляется в `Math_Hypothese_AutoCheck_Witch_Agents/`, если нужно:

- структурировать теоретические идеи;
- формулировать conjectures;
- исследовать инварианты, структуры и доказательные маршруты;
- запускать мультимодельный анализ;
- готовить formalization path.

## Фаза 5. Initial Research Package

После первичного исследования агент должен подготовить набор артефактов:

- competition brief;
- curated source list;
- список baseline ideas;
- список рисков и ограничений;
- initial hypotheses;
- предложение следующего набора экспериментов.

## Фаза 6. Experiment and Analysis Loop

Дальнейшая работа идёт циклом:

1. сформулировать гипотезу или baseline idea;
2. выбрать метод проверки;
3. провести эксперимент или теоретический анализ;
4. сохранить артефакты и summary;
5. обновить статусы и open questions.

## Фаза 7. Long-term Memory Update

После каждого существенного этапа агент обязан:

- обновить `RESEARCH_JOURNAL.md`;
- обновить профильные docs в дочернем проекте;
- при необходимости обновить `DOCUMENTATION_INDEX.md`;
- сохранить новые пользовательские уточнения в `USER_PROMPTS_LOG.md`.

## Критерии хорошей автономной работы

Хорошая автономная работа по Kaggle-задаче означает, что агент:

- не ограничивается одним baseline;
- умеет собрать и отфильтровать внешние источники;
- не теряет проблему, метрику и правила соревнования;
- отделяет engineering output от theory output;
- поддерживает длинную исследовательскую историю, а не только текущий diff.
